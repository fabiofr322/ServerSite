## 🔴 Prioridade Alta

### 1. Status Real do Servidor (Online/Offline)
**Situação atual:** O IP aparece como *"Servidor Offline (Desenvolvimento)"* com um ponto cinza fixo no código HTML.  
**Melhoria:** Integrar a [Minecraft Server Status API](https://api.mcsrvstat.us/) para mostrar em tempo real:
- 🟢 Jogadores online / Máximo de slots
- 🟡 Versão do servidor (ex: 1.21.4)
- 🔴 Offline com motivo

```js
// Exemplo de chamada
fetch('https://api.mcsrvstat.us/3/fr32survival.com')
  .then(r => r.json())
  .then(data => {
    if (data.online) showStatus(`${data.players.online}/${data.players.max} online`);
  });
```
**Impacto:** Jogadores veem em tempo real se o servidor está ativo antes de entrar.

## 🟡 Prioridade Média

### 6. Sistema de Conquistas / Badges
**Situação atual:** Rankings existem, mas sem gamificação visual.  
**Melhoria:** Criar sistema de badges automáticas baseadas em conquistas:
- 🏆 **Top Minerador** — 1º lugar no ranking de mineração por semana
- 💎 **Diamante** — 1 mês contínuo no servidor
- 🏗️ **Construtor** — submissão aprovada na galeria
- 🛡️ **Guardião** — usuário com 2FA ativo
- Exibição nos cards de veteranos e no perfil público

```sql
-- Tabela sugerida no Supabase
CREATE TABLE badges (
  id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id uuid REFERENCES public.profiles(id),
  badge_type text NOT NULL,  -- 'minerador', 'guardiao', etc.
  awarded_at timestamptz DEFAULT now()
);
```
